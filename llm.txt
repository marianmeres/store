## Purpose

Lightweight reactive store implementation with subscriptions and persistence support.
Svelte store contract compatible (https://svelte.dev/docs#component-format-script-4-prefix-stores-with-$-to-access-their-values-store-contract).

## Exported API

### Types

```typescript
type Subscribe<T> = (value: T) => void;
type Unsubscribe = () => void;
type Update<T> = (value: T) => T;

interface StoreReadable<T> {
    subscribe(cb: Subscribe<T>): Unsubscribe;
    get(): T;
}

interface StoreLike<T> extends StoreReadable<T> {
    set(value: T): void;
    update(cb: Update<T>): void;
}

interface CreateStoreOptions<T> {
    persist?: (v: T) => void;
    onPersistError?: (error: unknown) => void;
    onError?: (error: Error) => void;
}

interface CreateDerivedStoreOptions<T> extends CreateStoreOptions<T> {
    initialValue?: any;
}

interface Persistor<T> {
    remove: () => void;
    set: (v: T) => void;
    get: () => T | undefined;
    clear: () => void;
    __raw: () => any;
}
```

### Functions

1. `createStore<T>(initial?, options?): StoreLike<T>`
   - Creates a writable reactive store
   - Options: persist callback, onPersistError handler, onError handler
   - Uses strict equality (===) for change detection
   - Subscribers called synchronously on subscribe and on value change

2. `createDerivedStore<T>(stores, deriveFn, options?): StoreReadable<T>`
   - Creates a read-only derived store from source stores
   - deriveFn signature: `(values: any[], set?: CallableFunction) => T`
   - Supports sync (1 arg) and async (2 args with set callback) derivation
   - Lazy subscription: only subscribes to sources when derived has subscribers
   - get() triggers on-demand computation via temporary subscribe/unsubscribe

3. `createStoragePersistor<T>(key, type?): Persistor<T>`
   - Creates storage adapter for persistence
   - Types: "local" (localStorage), "session" (sessionStorage), "memory" (Map)
   - Default: "session"
   - JSON serialization for storage

4. `createStorageStore<T>(key, storageType?, initial?): StoreLike<T>`
   - Convenience: combines createStore + createStoragePersistor
   - Auto-persists on every change
   - Restores from storage on creation

5. `isStoreLike(value): boolean`
   - Duck typing check for subscribe method presence

## Implementation Details

### Store Internals
- Uses @marianmeres/pubsub for subscription management
- Publishes on "change" topic
- Persistence called on initial value and every set()
- set() only notifies if value !== previous (shallow comparison)

### Derived Store Internals
- Maintains internal subscription counter
- Subscribes to sources only when counter > 0
- Unsubscribes from sources when counter reaches 0
- deriveFn called once per source store on subscribe (causes duplicate calls)
- get() without active subscription: subscribes, gets value, immediately unsubscribes

### Memory Storage
- Uses module-level Map: `_memoryStorage`
- Shared across all memory persistors
- clear() affects all keys

## Testing

Tests cover:
- Store contract compliance (Svelte compatible)
- Subscription/unsubscription lifecycle
- set() shallow equality check (no notification for same value)
- update() callback pattern
- Derived sync and async derivation
- Derived lazy subscription behavior
- Persistence with error handling (onPersistError + default console.warn)
- Memory/storage persistor operations
- Edge cases: undefined, null, falsy values, NaN
- Type validation: function arguments required for subscribe/update/deriveFn
- Invalid storage type fallback

## Svelte Store Contract Compliance

1. subscribe() accepts subscription function
2. Subscription called immediately with current value
3. Returns unsubscribe function
4. All subscribers called synchronously on value change
5. Unsubscribe stops further notifications

## Common Patterns

### Basic Usage
```typescript
const store = createStore("initial");
const unsub = store.subscribe(console.log);
store.set("new value");
store.update(v => v + " updated");
unsub();
```

### Derived Store
```typescript
const a = createStore(1);
const b = createStore(2);
const sum = createDerivedStore([a, b], ([aVal, bVal]) => aVal + bVal);
```

### Async Derived
```typescript
const async = createDerivedStore([source], ([val], set) => {
    setTimeout(() => set(val * 2), 100);
});
```

### Persistent Store
```typescript
const persistor = createStoragePersistor("key", "local");
const store = createStore(persistor.get() ?? defaultValue, {
    persist: persistor.set
});
```

### Convenience Persistent Store
```typescript
const store = createStorageStore("key", "local", defaultValue);
```
